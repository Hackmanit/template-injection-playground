Template Injection Playground
Template Engines Config Links Random

With the Template Injection Playground a large number of the most relevant template engines can be tested for template injections.
For this purpose, simple web pages are provided, each of which uses one of the template engines.
Furthermore, various optional security measures such as sandboxes, encodings and denylists can be activated.

The Playground consists of the four subpages:

Template Engines

This page lists the implemented template engines. The template engines are assigned to their programming language and sorted in descending order of popularity.
Clicking on one of the template engines will take you to its overview page.
The overview page contains links to the documentation and the package manager website under Documentation.
The "Modes" section links to simple web pages that use the template engine. Most of them only have the "Default" mode. A few template engines also have a "Sandbox" or "Security" mode, if they offer such a thing.
In addition, most template engines have a cheat sheet that highlights special features and/or helpful template expressions.

Config

This page allows you to make configurations that apply to all websites with template engines. In general, errors can be caught, characters/words can be forbidden, or special characters can be HTML-encoded. Furthermore, it can be configured that template engines do not return their output in the response, but in a different place or not at all. Last but not least CSRF tokens can be activated.

Links

This page contains links to all web pages with template engines, including the different modes.
There is also a link to the echo web page, which shows the input, possibly modified due to the configuration. This is to test if scanners report a false positive for template injection.
It also links to the web page otherlocation, which - if set in the configuration - contains the output of the template engines. This is to test whether a scanner, after sending a template injection payload, will also look for the output in another specified location.

Random

This page selects a random template engine on the first call and embeds it as an iframe. This allows you to test yourself and your approach in a black box scenario.
Two buttons allow you to either select a new random template engine or reveal the template engine.
You can also use the URL /getRandom/ instead of the iframe.
